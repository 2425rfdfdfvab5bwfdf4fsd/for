ALGORITHMIC DEFINITIONS — SWING HIGH/LOW
================================================================================

SWING HIGH DEFINITION:
  A candle at index i is a swing high if:
    data['high'][i] > data['high'][i-n] for ALL n in [1..LOOKBACK]
    AND
    data['high'][i] > data['high'][i+n] for ALL n in [1..LOOKBACK]
  Where LOOKBACK = config.SWING_LOOKBACK (default: 5 candles each side)
  
  NOTE: Only completed candles can be swing points.
  The most recent LOOKBACK candles cannot yet be confirmed as swing highs.

SWING LOW DEFINITION:
  A candle at index i is a swing low if:
    data['low'][i] < data['low'][i-n] for ALL n in [1..LOOKBACK]
    AND
    data['low'][i] < data['low'][i+n] for ALL n in [1..LOOKBACK]

================================================================================
SWING CONFIRMATION RULE — CRITICAL (DECISION-017)
================================================================================

  EXACT CONFIRMATION ALGORITHM:
    Swing High at index i is confirmed when:
      ALL(data['high'][i] >= data['high'][j]
          for j in range(i - SWING_LOOKBACK_CANDLES, i + SWING_LOOKBACK_CANDLES + 1)
          if j != i)
    where SWING_LOOKBACK_CANDLES = config.SWING_LOOKBACK_CANDLES (default: 2)

    Swing Low at index i is confirmed when:
      ALL(data['low'][i] <= data['low'][j]
          for j in range(i - SWING_LOOKBACK_CANDLES, i + SWING_LOOKBACK_CANDLES + 1)
          if j != i)

  RULES (non-negotiable):
    1. Only CLOSED candles are used. The current forming candle (index 0 from
       the right) is NEVER included in swing detection.
    2. Minimum SWING_LOOKBACK_CANDLES = 1 (one candle required on each side).
    3. Default SWING_LOOKBACK_CANDLES = 2 (two candles each side for confirmation).
    4. This introduces a 2-candle lag — intentional trade-off to prevent look-ahead.
    5. The same value of SWING_LOOKBACK_CANDLES MUST be used in both live trading
       and backtesting to ensure consistent strategy behavior.
    6. Configurable via SWING_LOOKBACK_CANDLES in app/config.py.

  WHY THIS MATTERS:
    Without a defined N, each agent will choose a different lookback, making
    strategy behavior non-deterministic and backtesting non-reproducible.
    An N=0 value introduces look-ahead bias (using an unconfirmed swing).

  NOTE: The existing SWING_LOOKBACK=20 parameter in config is the SCAN WINDOW
  (how far back to look for swings in the data). SWING_LOOKBACK_CANDLES=2 is
  the CONFIRMATION window (how many candles each side must be lower/higher).
  These are separate parameters serving different purposes.

TREND DETERMINATION:
  Given a sequence of confirmed swing highs (SH) and swing lows (SL):
  
  BULLISH TREND:
    Current SH > Previous SH (Higher High)
    AND Current SL > Previous SL (Higher Low)
  
  BEARISH TREND:
    Current SH < Previous SH (Lower High)
    AND Current SL < Previous SL (Lower Low)
  
  RANGING/UNCLEAR:
    Mixed — not fitting bullish or bearish criteria
