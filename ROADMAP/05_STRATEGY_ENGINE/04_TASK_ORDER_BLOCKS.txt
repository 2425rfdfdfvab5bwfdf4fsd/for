ALGORITHMIC DEFINITIONS — ORDER BLOCK
================================================================================

BULLISH ORDER BLOCK (demand zone — support for BUY entries):
  Definition:
    The last BEARISH candle (close < open) immediately before a
    significant bullish move (displacement). This bearish candle
    is the OB.
  
  Exact Detection:
  1. Find displacement move (see Phase 05-06)
  2. Look back from the displacement start
  3. Find the last candle where close < open (bearish candle)
  4. That candle's body defines the OB zone:
       OB_high = max(open, close) of the bearish candle
       OB_low  = min(open, close) of the bearish candle
     (Some practitioners use the candle's full range — wick included.
      Use BODY only for stricter definition.)
  
  ALTERNATIVE (simpler, more common):
    OB_high = candle high
    OB_low  = candle low
    (Include wicks for wider zone)

BEARISH ORDER BLOCK (supply zone — resistance for SELL entries):
  Definition:
    The last BULLISH candle (close > open) before a significant
    bearish move.
  
  Detection:
  1. Find bearish displacement
  2. Last candle where close > open before the displacement
  3. OB zone:
       OB_high = candle high
       OB_low  = candle low

ORDER BLOCK FRESHNESS:
  An OB is FRESH if price has not yet returned into the OB zone.
  An OB is MITIGATED if price has entered the zone (closed within the OB).
  An OB is INVALIDATED if price has CLOSED beyond the OB:
    Bullish OB invalidated if close < OB_low
    Bearish OB invalidated if close > OB_high

ORDER BLOCK VALIDITY RULES:
  1. Must be formed after a BOS in the same direction
  2. Must be FRESH (not yet mitigated)
  3. Must be within the last N candles (max_age = 50 by default)
  4. The displacement move must be significant (see displacement task)

================================================================================
ORDER_BLOCK_EXACT_DEFINITION — REQUIRED (CHG-012)
================================================================================

BULLISH ORDER BLOCK:
  - The LAST bearish candle (close < open) immediately preceding a bullish
    BOS impulse move (displacement). "Last" means the candle directly before
    the displacement sequence starts.
  - OB zone = [candle.low, candle.high] (full range including wicks)
  - If using body-only: OB zone = [min(open,close), max(open,close)]
  - DEFAULT: use FULL RANGE (wicks included) for wider, more reliable zone.
  - CONFIG: OB_USE_BODY_ONLY = false (default)

BEARISH ORDER BLOCK:
  - The LAST bullish candle (close > open) immediately preceding a bearish
    BOS impulse move. OB zone = [candle.low, candle.high]

FRESHNESS STATES (exact definitions):
  FRESH:      price has NOT re-entered the OB zone [low, high] since OB formed.
              Test: ALL candles after OB formation have high < OB.high (for bullish)
              OR low > OB.low (for bearish)
  MITIGATED:  price has entered the zone but NOT closed beyond the far edge.
              Test: any candle after OB has low <= OB.high (for bullish OB touched)
              AND no candle has close < OB.low
  INVALIDATED: price has CLOSED beyond the far edge of the OB zone.
              Test: any candle has close < OB.low (for bullish OB — invalidated)
              OR close > OB.high (for bearish OB — invalidated)

TIMEFRAME USAGE RULES:
  M15 and H1 OBs: used as ENTRY ZONES (price must return to OB for entry)
  H4 OBs:         used for DIRECTIONAL BIAS ONLY (not entry zones)

VALIDITY RULE: An OB is only valid if:
  1. It was formed after a confirmed BOS in the same direction
  2. It is currently FRESH (not mitigated or invalidated)
  3. Age <= OB_MAX_AGE_CANDLES (default: 50 M15 candles)
  4. The displacement move that created it was significant (>= 1.5x ATR)
