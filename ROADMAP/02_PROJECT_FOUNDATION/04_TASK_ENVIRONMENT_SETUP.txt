# MT5 Automated Forex Trading Bot

## Project Summary
Fully automated Forex trading system that connects to MetaTrader 5 on Windows,
scans EURUSD/GBPUSD/USDJPY using a deterministic SMC/ICT strategy across
H4/H1/M15/M5 timeframes, and executes 0–3 high-quality trades per day.
$0 software cost. No LLM APIs. Runs 24/5 during London + New York sessions.

## Tech Stack
- Python 3.11+
- MetaTrader5 (Windows Python package)
- FastAPI + HTML/CSS/JS (local dashboard)
- SQLite (local persistence)
- python-dotenv (.env configuration)
- pytest (testing)

## Architecture Pattern
Layered modules: mt5/ → strategy/ → confluence/ → risk/ → execution/ → management/
All modules import config from app/config.py
All modules use loggers from app/logger.py
Database access via app/database/repositories.py only

## How to Run (Windows)
  setup.bat          — First-time setup
  start_bot.bat      — Start the trading bot
  run_dashboard.bat  — Start the web dashboard
  run_tests.bat      — Run all tests

## Critical Rules for Agent
1. NEVER modify files outside the scope stated in the task
2. NEVER add packages to requirements.txt without explicit permission
3. NEVER hardcode values — use app/config.py
4. NEVER use print() — use logger from app/logger.py
5. NEVER enable live trading (LIVE_TRADING must stay false during development)
6. ALWAYS mock MetaTrader5 in all tests (MT5 is Windows-only)
7. ALWAYS follow patterns in CODE_STANDARDS.md
8. ALWAYS write tests for new business logic

## Default Mode
TRADING_MODE=DEMO, LIVE_TRADING=false — NEVER change this default

      result = risky_operation()

      return None  # or raise depending on criticality
  except Exception as e:
      logger.critical(f"Unexpected error in {context}: {e}", exc_info=True)

## Trading Safety Pattern

  if not risk_manager.can_trade(signal):

DETAILED REQUIREMENTS — TRADING_RULES.md
================================================================================

Document ALL trading rules that must be followed:
  - Trade only during London (07:00–16:00 UTC) or New York (12:00–21:00 UTC)
  - Maximum 3 trades per day
  - Minimum 0 trades — NEVER force a trade
  - Risk 0.5% of equity per trade
  - Minimum confluence score: 8/10
  - Minimum R:R: 1:2
  - Daily loss limit: 2%
  - Consecutive loss protection: 2 losses = stop for the day
  - No overnight positions by default
  - No weekend exposure
  - News blackout: 30 minutes before/after high-impact events
  - Spread check required before every entry
  - Correlation check required before every entry
  - The 55–65% win rate is a PERFORMANCE TARGET, not a guarantee

================================================================================
DETAILED REQUIREMENTS — RISK_MANAGEMENT.md
================================================================================

Document the exact risk management rules:
  - Position sizing formula: lots = (equity * risk%) / (SL_pips * pip_value)
  - SL = structure-based + ATR buffer
  - TP = structure/liquidity target that gives min 1:2 R:R
  - Daily risk state persists in SQLite across restarts
  - Daily trade count persists in SQLite across restarts
  - Consecutive losses persist in SQLite across restarts
