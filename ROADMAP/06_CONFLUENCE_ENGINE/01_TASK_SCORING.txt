OBJECTIVE:
    Implement the ConfluenceScorer class that evaluates a raw signal against a
    defined set of weighted factors and returns a ScoredSignal with a numeric
    score (0–10) and a per-factor breakdown.

CONTEXT:
    Every trade candidate produced by the Strategy Engine must pass through this
    scorer. The score is calculated deterministically from measurable conditions.
    No vague criteria. Every factor has a binary (present/absent) or graded value
    that contributes a fixed weight to the total.

    Minimum passing score: 8/10 (configurable via MIN_CONFLUENCE_SCORE).

    The 55–65% win-rate target is NOT guaranteed. The scorer maximises setup
    quality, not trade frequency.

DETAILED REQUIREMENTS:
--------------------------------------------------------------------------------

SCORING FACTORS (10 factors, total weight = 10.0):

    1. H4 TREND ALIGNMENT
       Condition: Signal direction aligns with H4 trend (EMA trend filter)
       Weight: 1.0

    2. H1 STRUCTURE CONFIRMATION
       Condition: H1 shows BOS or CHoCH in signal direction
       Weight: 1.0

    3. ORDER BLOCK (combined: present AND fresh)
       Condition: Valid, unmitigated Order Block exists at entry zone on M15/H1.
                  Freshness (price has not re-entered OB zone) is a required
                  sub-condition — not a separate score entry.
       Weight: 1.0  (binary: OB present AND fresh = 1.0; absent or stale = 0.0)

    4. FAIR VALUE GAP (FVG)
       Condition: An unmitigated FVG overlaps the entry zone
       Weight: 1.0

    5. LIQUIDITY SWEEP
       Condition: A recent liquidity sweep (equal highs/lows swept) precedes the setup
       Weight: 1.0

    6. DISPLACEMENT CANDLE
       Condition: A displacement candle (strong impulsive move) created the setup
       Weight: 1.0

    7. HTF ORDER BLOCK CONFLUENCE
       Condition: An unmitigated H1 or H4 Order Block exists at the current price
                  level — tests whether multiple timeframes agree on the OB zone.
                  REPLACES Session Alignment (session is already enforced as a
                  mandatory filter gate; scoring it gives a redundant free point).
       Weight: 1.0

    8. ATR VOLATILITY ACCEPTABLE
       Condition: Current ATR is within configured min/max volatility bounds
       Weight: 0.5

    9. SPREAD ACCEPTABLE
       Condition: Current spread is below MAX_SPREAD_PIPS for the symbol
       Weight: 0.5

   10. M5 ENTRY CONFIRMATION
       Condition: M5 shows micro-structure alignment (e.g. bullish engulf / M5 BOS)
       Weight: 1.0

   TOTAL POSSIBLE: 10.0  (1+1+1+1+1+1+1+0.5+0.5+1 = 10.0 — verified correct)

   NOTE: Weights are stored in app/config.py so they can be audited and
   adjusted without modifying scorer.py logic.

   CHANGES FROM ORIGINAL DESIGN:
     DECISION-016: ORDER_BLOCK_PRESENT(1.0)+ORDER_BLOCK_FRESH(0.5) merged into
                   single ORDER_BLOCK(1.0). Total was 10.5 — now correctly 10.0.
     DECISION-018: SESSION_ALIGNMENT replaced by HTF_OB_CONFLUENCE. Session is
                   a mandatory filter gate, not a quality differentiator.
