OBJECTIVE:
    Implement OrphanPositionRecovery that detects MT5 positions that exist
    without a corresponding database record (orphans) and handles them safely.

CONTEXT:
    After a crash or unexpected restart, MT5 may have positions that were
    placed before the database was updated. These orphan positions must be:
    1. Detected on startup
    2. Logged with CRITICAL priority
    3. Either adopted (added to database) or flagged for manual review
    4. Never silently left unmanaged

    ORPHAN_RECOVERY_POLICY (CHG-016 — configurable via ORPHAN_POLICY in .env):

    DEFAULT: 'alert'
      - Do NOT auto-close the orphan position
      - Log as CRITICAL immediately
      - Send Telegram alert (if enabled)
      - Record in database as status='untracked'
      - Operator must manually resolve via MT5 terminal
      - NEVER silently ignore an orphan

    'close': auto-close at market price
      - Close the orphan via mt5.order_send() with SELL/BUY to close
      - Log full details of the forced close
      - Record as 'force_closed' in database
      - Send Telegram alert

    'adopt': adopt into bot management
      - Reconstruct parameters from MT5 position data (best-effort)
      - Insert into trades table with reconstructed SL/TP from position
      - Bot will now manage this position normally
      - Log WARNING that parameters are reconstructed (not original)
      - Only use if you trust the position was placed by this bot

    .env.example ADDITION:
      # ORPHAN_POLICY: how to handle positions in MT5 with no DB record
      # Options: alert (default), close, adopt
      ORPHAN_POLICY=alert
