EVIDENCE GATE THRESHOLD CLARIFICATION — REQUIRED (CHG-C06)
================================================================================

The master roadmap states: "min 20 trades total, 8 trades/segment"
This task file states: MINIMUM_SEGMENT_SAMPLE = 15

These are NOT contradictory — they apply at different evaluation levels:

  GLOBAL GATE (applied before any segment analysis begins):
    Minimum 20 total CLOSED trades across ALL segments combined.
    If total_closed_trades < 20: suppress ALL recommendations.
    Reason: insufficient overall sample for any reliable inference.

  SEGMENT GATE (applied per-segment when global gate passes):
    Minimum 8 closed trades within the SPECIFIC segment being analyzed.
    (e.g., EURUSD trades, or London-session trades, or OB-setup trades)
    If segment_trades < 8: suppress recommendation for THAT segment only.

  STATISTICAL GATE (applied after segment gate):
    MINIMUM_SEGMENT_SAMPLE = 15 applies to the STATISTICAL SIGNIFICANCE check.
    A segment needs >= 15 trades before a win-rate confidence interval is
    computed (normal approximation requires n >= 15 for adequate accuracy).
    Below 15: evidence_level = SUGGESTIVE (surfaced but flagged as low-confidence).

SUMMARY:
  20 total trades    → unlock global analysis
   8 per segment     → unlock segment-level analysis
  15 per segment     → unlock statistical significance calculation

================================================================================
    1. MINIMUM_SAMPLE: segment must have >= MINIMUM_SEGMENT_SAMPLE trades (default: 15)
    2. MINIMUM_PERIOD: data must span >= MINIMUM_PERIOD_DAYS (default: 30)
    3. CONSISTENCY: the finding must be consistent across at least 2 separate weeks
    4. NOT_CHERRY_PICKED: the comparison group must also have adequate sample
    5. STATISTICAL_SIGNIFICANCE: win rate difference >= 10 percentage points
