SERVER_TIMEZONE_DETECTION — REQUIRED (CHG-008)
================================================================================

The broker server time is NOT the same as UTC. Most MT5 brokers use UTC+2
(or UTC+3 during DST). Using the wrong timezone will cause the daily loss
limit to reset at the wrong time, potentially allowing excess risk.

DETECTION ALGORITHM (call on startup):
  1. Query mt5.symbol_info_tick(symbol) to get server_time (MT5 timestamp)
  2. Compare server_time with datetime.utcnow() to compute offset
  3. Round to nearest hour: SERVER_UTC_OFFSET_HOURS = round((server_time - utc_now).seconds / 3600)
  4. Store in config (or pass to DailyLimitsChecker)
  5. Log: "Detected broker server UTC offset: +{X} hours"
  6. Validate against .env configured SERVER_UTC_OFFSET_HOURS if set

.env.example ADDITION:
  # Broker server timezone offset from UTC (most MT5 brokers: +2 or +3)
  SERVER_UTC_OFFSET_HOURS=2

  # Server DST behavior (brokers often shift by 1 hour for US/EU DST)
  # Check broker documentation — most use UTC+2 in summer, UTC+3 in winter
  # or vice versa. Set this to the offset that matches your broker's daily reset.

IMPLEMENTATION NOTE:
