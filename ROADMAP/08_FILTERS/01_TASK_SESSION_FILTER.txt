DST_HANDLING — REQUIRED (CHG-019)
================================================================================

London and New York sessions shift by 1 hour during DST transitions.
Without explicit handling, the bot trades at wrong hours twice per year.

SESSION HOURS IN UTC (use these — never use local system clock):

  LONDON SESSION:
    Summer (BST, UTC+1): 07:00–16:00 UTC
    Winter (GMT, UTC+0): 08:00–17:00 UTC

  NEW YORK SESSION:
    Summer (EDT, UTC-4): 13:00–22:00 UTC
    Winter (EST, UTC-5): 14:00–23:00 UTC

  NOTE: The session window CONFIG parameters (LONDON_START_UTC etc.) should
  reflect the SUMMER hours as default. DST detection adjusts them automatically.

IMPLEMENTATION:
  Use Python's stdlib zoneinfo module (Python 3.9+, no external dependency):
    from zoneinfo import ZoneInfo
    from datetime import datetime, timezone

    def is_london_session(utc_dt: datetime) -> bool:
        # Convert to London timezone to get correct DST-aware hour
        london_tz = ZoneInfo("Europe/London")
        london_time = utc_dt.astimezone(london_tz)
        return datetime(2000,1,1, LONDON_START_HOUR) <= ...
        # OR: use UTC hours directly based on DST status of the London zone

  SIMPLER APPROACH (recommended):
    # Determine if US/EU is in DST using tz offset
    london_tz = ZoneInfo("Europe/London")
    ny_tz = ZoneInfo("America/New_York")
    london_offset = datetime.now(london_tz).utcoffset().total_seconds() / 3600
    # If london_offset == 1.0: summer (BST) — use 07:00–16:00 UTC
    # If london_offset == 0.0: winter (GMT) — use 08:00–17:00 UTC

ALWAYS USE:
    current_utc = datetime.now(timezone.utc)  # timezone-aware UTC datetime
    NEVER use: datetime.utcnow()              # naive, no DST context
