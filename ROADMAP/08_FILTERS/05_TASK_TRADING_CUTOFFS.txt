PHASE:       08 — Filters
================================================================================

OBJECTIVE:
    Implement TradingCutoffFilter that enforces end-of-day, overnight, and
    weekend exposure rules to prevent uncontrolled position exposure.

CONTEXT:
    CUTOFF RULES:
    1. END-OF-DAY CUTOFF: Stop opening new trades after EOD_CUTOFF_UTC (default 19:30)
    2. OVERNIGHT CUTOFF: If OVERNIGHT_TRADING_ENABLED=false (default), close all
       open positions before END_OF_DAY_CLOSE_UTC (default 20:00)
    3. WEEKEND CUTOFF: No new trades after FRIDAY_CUTOFF_UTC (default Friday 19:00)
       All positions closed before FRIDAY_CLOSE_UTC (default Friday 19:30)
    4. MONDAY OPEN: No new trades until MONDAY_OPEN_UTC (default Monday 07:00)

    NOTE: Closing of positions on cutoff is handled by Position Management
    (Phase 10). This filter only blocks NEW trade entry.

DEPENDENCIES:
    - Phase 02 config
    - Session filter (Phase 08-01)

FILES TO CREATE:
    app/filters/trading_cutoffs.py
    tests/test_filters/test_cutoffs.py

FILES FORBIDDEN TO MODIFY:
    app/mt5/
    app/risk/
    app/strategy/
    ROADMAP/

DETAILED REQUIREMENTS:
    class TradingCutoffFilter:
        check(utc_datetime: datetime) -> FilterResult

    Checks (in order):
        1. Is it Friday after FRIDAY_CUTOFF_UTC? → BLOCK "FRIDAY_CUTOFF"
        2. Is it Saturday or Sunday? → BLOCK "WEEKEND"
        3. Is it Monday before MONDAY_OPEN_UTC? → BLOCK "MONDAY_PRE_OPEN"
        4. Is it after EOD_CUTOFF_UTC on weekday? → BLOCK "EOD_CUTOFF"
        else: PASS

TEST REQUIREMENTS:
